# Dressify Deployment Guide - 500 Error Fixes

## Issues Fixed for Render Deployment

### 1. **Database Configuration**
- ❌ **Problem**: Environment variables (DB_NAME, DB_USER, etc.) were not properly defaulted
- ✅ **Fix**: Added default values in `settings.py` for PostgreSQL database connections
- **Location**: `dressify/settings.py` - DATABASE section

### 2. **ALLOWED_HOSTS Configuration**
- ❌ **Problem**: Set to `['*']` which can cause issues in production
- ✅ **Fix**: Changed to properly include Render domain: `[RENDERED_DOMAIN, "localhost", "127.0.0.1", "*.onrender.com"]`
- **Location**: `dressify/settings.py` - ALLOWED_HOSTS section

### 3. **Static Files Storage**
- ❌ **Problem**: Using `CompressedManifestStaticFilesStorage` can fail during build
- ✅ **Fix**: Use simple `StaticFilesStorage` on Render, compressed only for local
- **Location**: `dressify/settings.py` - STATICFILES_STORAGE section

### 4. **Missing Logging Configuration**
- ❌ **Problem**: No logging setup to debug 500 errors
- ✅ **Fix**: Added comprehensive logging configuration to track errors
- **Location**: `dressify/settings.py` - LOGGING section (bottom)

### 5. **Build Script Improvements**
- ❌ **Problem**: Build would fail on first run due to missing directories
- ✅ **Fix**: Create media directories and handle migration errors gracefully
- **Location**: `build.sh`

### 6. **URL Configuration**
- ❌ **Problem**: Missing admin panel and media serving setup
- ✅ **Fix**: Added Django admin and conditional media serving
- **Location**: `dressify/urls.py`

## Deployment Environment Variables Required

Set these in Render dashboard under Environment Variables:

```
SECRET_KEY=<auto-generated>
DB_NAME=dressify_db
DB_USER=<auto-from-database>
DB_PASSWORD=<auto-from-database>
DB_HOST=<auto-from-database>
DB_PORT=5432
RENDER_EXTERNAL_HOSTNAME=<your-app>.onrender.com
```

## Database Setup

1. Create a PostgreSQL database on Render
2. Run migrations:
   ```bash
   python manage.py migrate
   ```
3. Create admin user:
   ```bash
   python manage.py createsuperuser
   ```

## Troubleshooting 500 Errors

### Check Logs on Render:
1. Go to your Render service dashboard
2. Click "Logs" tab
3. Look for error messages

### Common Issues:

| Error | Solution |
|-------|----------|
| `ProgrammingError: relation "app_user" does not exist` | Run migrations: `python manage.py migrate` |
| `ALLOWED_HOSTS` error | Check `RENDER_EXTERNAL_HOSTNAME` env var is set |
| `static files not found (404)` | Run: `python manage.py collectstatic` |
| `database connection refused` | Check DB credentials in environment variables |
| `no such table: app_user` | Database not migrated - rerun build |

## Local Testing

1. Create `.env` file (copy from `.env.example`)
2. Set `DEBUG=true`
3. Run locally: `python manage.py runserver`
4. Test all endpoints before deploying
