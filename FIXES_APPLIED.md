## Deployment Checklist

Before deploying to Render:

### 1. **Set Environment Variables in Render Dashboard:**
```
DEBUG=false
SECRET_KEY=<auto-generated>
DJANGO_SETTINGS_MODULE=dressify.settings
```

Database credentials will be auto-configured by Render.

### 2. **Create Database Service** (if not auto-created)
- Name: `dressify-db`
- Type: PostgreSQL
- Version: 14
- Plan: Free

### 3. **Create Web Service**
- Name: `dressify`
- Runtime: Python
- Build Command: `bash build.sh`
- Start Command: `gunicorn dressify.wsgi:application --bind 0.0.0.0:$PORT --workers 2`

### 4. **Common 500 Errors & Solutions**

| Error | Solution |
|-------|----------|
| `ProgrammingError: relation "app_user" does not exist` | Migrations didn't run - check build logs |
| `could not connect to server` | Database credentials not set - verify env vars |
| `ALLOWED_HOSTS` error | Make sure `RENDER_EXTERNAL_HOSTNAME` is set |
| `Static files 404` | Check `staticfiles` directory exists |
| `connection timeout` | Increase timeout in settings.py |

## Verify Production Setup

1. **Check Render Logs:**
   - Go to your service dashboard
   - Click "Logs" tab
   - Look for `Build completed successfully`

2. **Test Your App:**
   - Visit `https://your-app.onrender.com`
   - Check admin panel: `/admin/`
   - Try logging in and placing an order

3. **If Still Getting 500 Errors:**
   - Check Render logs for specific error
   - Run migrations manually if needed:
   ```bash
   render run python manage.py migrate
   ```
   - Create superuser:
   ```bash
   render run python manage.py createsuperuser
   ```
